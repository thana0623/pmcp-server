读取项目记忆文件，总结当前状态，建议下一步操作。

## 步骤

1. 读取以下文件（按顺序）：
   - `HOT_STATE.md` — 上一个会话的热状态（最重要）
   - `.github/prompts/context.md` — 项目上下文（按需读，太长）

2. 如果 HOT_STATE.md 有内容（"最后更新"不是"待填写"），用它恢复上下文：
   - 告诉用户：**上次会话在做什么 + 最近讨论方向 + 下一步 + 阻塞点**

3. 用一句话总结：**项目是什么 + 当前在做什么 + 上次做了什么**

4. 根据 HOT_STATE.md 内容建议下一步：

| 状态 | 建议 |
|------|------|
| 没有活跃任务 | "没有活跃任务。想做什么？" |
| 有任务进行中 | "上次在做 XXX，继续吗？" |
| 有阻塞点 | "上次卡在 XXX，要解决吗？" |

5. 列出可用命令（只在首次启动时显示）：

```
可用命令：/start /end /loop /commit /push
```

## 注意

- 如果文件不存在，运行 `pmcp setup` 初始化
- 不要输出文件原文，只输出总结和建议
- 保持简洁，3-5 行足够
- HOT_STATE.md 是最重要的——它包含上一个会话的讨论上下文
