读取会话状态，恢复上下文，建议下一步。

## 步骤

### 1. 读取当前会话文件
读取 `.github/prompts/active-session.md`：
- 如果有实际内容（不是"待填写"）→ 这是上次未归档的进度，直接恢复
- 如果是空的/全是"待填写" → 继续步骤 2

### 2. 读取上次归档
读取 `.github/prompts/sessions.md`，找到最后一个 `## 会话归档` 区块：
- 提取：做了什么、关键决策、阻塞点
- 这是上次会话的完整上下文

### 3. 输出总结
格式：
```
上次会话：{一句话总结做了什么}
关键决策：{1-2 个重要决策}
当前状态：{进行中/已完成/有阻塞}
```

### 4. 建议下一步
- 如果上次有未完成的任务 → "上次任务未完成，继续？还是开始新任务？"
- 如果上次已完成 → "没有活跃任务。想做什么？"
- 如果有阻塞 → "有阻塞：{问题}，先解决这个？"

### 5. 更新 active-session.md
用 Edit 工具更新 `.github/prompts/active-session.md`：
- 当前任务：恢复上次的任务描述（或"待定"）
- 进展：从归档中提取
- 关键决策：从归档中提取

## 注意
- 不要输出文件原文，只输出总结
- 保持简洁，5 行以内
- 如果没有任何历史，输出"首次启动，没有历史记录"
